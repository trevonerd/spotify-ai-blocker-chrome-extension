## Critical Bug Found and Fixed

### Bug: UI Freeze in `getNowPlaying()`

**Discovery**: During synthetic DOM testing, identified that `getNowPlaying()` in `page-script.ts` used a synchronous busy-wait loop:

```typescript
// BROKEN - blocked main thread for up to 5 seconds
const waitUntil = Date.now() + interval;
while (Date.now() < waitUntil) { /* busy wait */ }
```

**Impact**: HIGH — User-facing freeze during context menu "Report AI Artist" usage. Spotify's UI would become unresponsive for up to 5 seconds.

**Fix Applied**: Converted to async Promise-based retry with `setTimeout` polling:

```typescript
// FIXED - non-blocking async retry
async function getNowPlaying(): Promise<NowPlayingInfo> {
  return new Promise((resolve) => {
    // ... selectors ...
    function tryQuery(): void {
      // ... query logic ...
      setTimeout(tryQuery, intervalMs);  // Non-blocking
    }
    tryQuery();
  });
}
```

**Verification**: Synthetic tests confirmed both race condition and static scenarios now work correctly without blocking.
